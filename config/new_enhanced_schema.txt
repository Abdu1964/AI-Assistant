#### Important Rules:
1. **Database ID vs Property Value:**
   - `id` field: ONLY use for actual database identifiers (lowercase, like "ensg00000290825")
   - `properties`: Use for searchable values (like gene names, transcript IDs, exon numbers)

2. **When to Use Each:**
   - Query by database ID: Put the ID in `id` field, leave `properties` empty
   - Query by name/identifier: Put the value in `properties`, leave `id` empty

3. **Examples:**
   - "Find gene BRCA1" â†’ `properties: {"gene_name": "BRCA1"}`, `id: ""`
   - "Find gene with ID ensg00000186092" â†’ `id: "ensg00000186092"`, `properties: {}`
   - "Find transcript ENST00000441515" â†’ `properties: {"transcript_id": "ENST00000441515"}`, `id: ""`

#### Notes:
- All relationships are directional and have no properties
- Gene types include: lncRNA, protein_coding, miRNA, pseudogene, snRNA, etc.
- Transcript types include: lncRNA, protein_coding, miRNA, pseudogene, processed_transcript, etc.
- Exons are numbered and belong to specific transcripts
- Genes can have multiple transcripts, and transcripts can have multiple exons
