### Schema

#### Nodes:
Node properties:
gene {id: STRING, label: STRING, gene_type: STRING, chr: STRING, start: STRING, end: STRING, gene_name: STRING, synonyms: STRING}
transcript {id: STRING, label: STRING, transcript_id: STRING, transcript_name: STRING, transcript_type: STRING, chr: STRING, start: STRING, end: STRING, gene_name: STRING}
exon {id: STRING, label: STRING, gene_id: STRING, exon_id: STRING, exon_number: STRING, chr: STRING, transcript_id: STRING, start: STRING, end: STRING}

#### Relationships:
(:gene)-[:transcribed_to]->(:transcript)
(:transcript)-[:transcribed_from]->(:gene)
(:transcript)-[:includes]->(:exon)

#### Important Rules:
1. **Database ID vs Property Value:**
   - `id` field: ONLY use for actual database identifiers (lowercase)
   - `properties`: Use for searchable values (names, IDs, numbers)

2. **When to Use Each:**
   - Query by database ID: Put the ID in `id` field, leave `properties` empty
   - Query by name/identifier: Put the value in `properties`, leave `id` empty

#### Notes:
- All relationships are directional and have no properties
- Gene types include: lncRNA, protein_coding, miRNA, pseudogene, snRNA, etc.
- Transcript types include: lncRNA, protein_coding, miRNA, pseudogene, processed_transcript, etc.
- Exons are numbered and belong to specific transcripts
- Genes can have multiple transcripts, and transcripts can have multiple exons
